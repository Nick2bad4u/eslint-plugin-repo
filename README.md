# eslint-plugin-repo

ESLint plugin for enforcing repository-compliance standards across hosting providers such as GitHub, GitLab, Bitbucket, and Codeberg/Forgejo.

## Installation

```sh
npm install --save-dev eslint-plugin-repo typescript
```

## Quick start (Flat Config)

```js
import repoPlugin from "eslint-plugin-repo";

export default [repoPlugin.configs.recommended];
```

## Presets

- `repoPlugin.configs.recommended` — cross-provider baseline repository hygiene.
- `repoPlugin.configs.strict` — recommended plus stronger policy checks.
- `repoPlugin.configs.github` — GitHub-specific repository requirements.
- `repoPlugin.configs.gitlab` — GitLab-specific repository requirements.
- `repoPlugin.configs.bitbucket` — Bitbucket-specific repository requirements.
- `repoPlugin.configs.codeberg` — Codeberg/Forgejo-specific repository requirements.
- `repoPlugin.configs.ai` — AI-assistance repository guidance requirements.
- `repoPlugin.configs.all` — union of all current rules.

See detailed preset pages under [`docs/rules/presets`](./docs/rules/presets/).

## Complementary GitHub Actions plugin

If you want stricter GitHub Actions/Dependabot policy checks, use [eslint-plugin-github-actions-2](https://github.com/Nick2bad4u/eslint-plugin-github-actions-2) alongside this plugin.

This plugin (`eslint-plugin-repo`) focuses on cross-provider repository compliance, while `eslint-plugin-github-actions-2` provides deeper GitHub Actions and Dependabot workflow hardening.
